# 05 深入理解 LLM 原理

## 01 LLM 原理学习思路

学习 LLM 原理，不建议一上来就死磕论文、公式和各种模型名。

更好的方式是先建立一条主线：

```text
文本输入
  ↓
分词与向量表示
  ↓
神经网络计算
  ↓
参数和权重参与判断
  ↓
训练时通过损失函数、反向传播、梯度下降更新参数
  ↓
推理时根据上下文预测下一个 token
  ↓
连续预测，生成完整回答
```

只要这条主线能想清楚，后面再看 Transformer、自回归、上下文窗口、指令微调、RAG、多模态，就不会觉得每个概念都是孤立的。
 
### 1. 先明确学习目标

学习 LLM 原理的目的，不是把自己变成算法科学家，而是为了能更好地理解 AI 应用开发中的关键问题：

- 为什么模型能理解语义？
- 为什么模型有时会胡说？
- 为什么上下文太长会影响效果？
- 为什么参数更多通常能力更强，但不一定永远更好？
- 为什么训练需要大量 GPU？
- 为什么 RAG 不是训练模型，却能提升知识问答效果？
- 为什么 Agent、工具调用、长上下文、本地模型部署都会受到模型原理影响？

所以这门课的重点不是“会不会推导公式”，而是：

```text
能不能从工程视角理解大模型到底在做什么。
```

### 2. 不要把 LLM 当成黑盒聊天工具

如果只从使用者角度看，LLM 很像一个聊天机器人：

```text
我输入一句话，它输出一段回答。
```

但从原理角度看，它更接近一个巨大的概率预测系统：

```text
根据已有上下文，预测下一个最可能出现的 token。
```

比如用户问：

```text
床前明月光，下一句是什么？
```

模型不是像人一样“打开记忆库查古诗”，而是根据训练中见过的大量文本模式，计算当前上下文后面最可能接什么 token。因为“床前明月光”后面高度对应“疑是地上霜”，所以模型会生成这个结果。

这也是理解 LLM 的第一个关键：

```text
LLM 表面上是在回答问题，本质上是在做上下文条件下的下一个 token 预测。
```

### 3. 第一阶段：理解语义如何变成数字

计算机不能直接理解文字，它只能处理数字。

所以学习 LLM 的第一步，是理解文字如何变成模型可以计算的形式。

大致过程是：

```text
一句话
  ↓
切成 token
  ↓
每个 token 变成向量
  ↓
向量进入神经网络参与计算
```

可以先掌握几个概念：

| 概念 | 先理解成什么 |
| --- | --- |
| token | 模型处理文本的基本单位，可以是字、词、词片段或符号 |
| embedding | 把 token 表示成一组数字，也就是向量 |
| 向量空间 | 语义相近的内容，在数学空间里距离更近 |
| 语义表示 | 模型不是直接懂文字，而是通过向量关系捕捉语义 |

这一阶段要解决的问题是：

```text
为什么“猫”和“狗”会比“猫”和“冰箱”更接近？
```

因为模型会把文字映射成向量，而这些向量在训练过程中逐渐学到语义关系。

### 4. 第二阶段：理解神经网络如何学习

理解向量之后，下一步是理解神经网络是怎么“学会”的。

这里不需要一开始就掌握复杂数学，但要抓住几个核心概念：

```text
参数
权重
损失函数
反向传播
梯度下降
过拟合
调参
```

可以这样理解：

```text
参数和权重：模型内部用来做判断的可调整数字
损失函数：衡量模型预测错得有多离谱
反向传播：找到哪些参数导致了错误
梯度下降：沿着让错误变小的方向调整参数
训练：不断重复预测、计算误差、调整参数的过程
```

一句话概括：

```text
训练不是把知识直接塞进模型，而是让模型通过大量样本不断调整参数。
```

这也能解释为什么数据标注本身不是反向传播。

数据标注提供的是训练样本和参考答案，反向传播是模型根据预测误差更新参数的数学过程。一个是准备训练材料，一个是模型内部学习机制。

### 5. 第三阶段：理解 LLM 的训练目标

主流 GPT 类模型常用的核心训练目标是：

```text
根据前面的 token，预测下一个 token。
```

这通常叫：

```text
Next Token Prediction，简称 NTP。
```

例如训练样本是：

```text
我今天想吃火锅
```

模型会学习：

```text
看到“我”，预测“今天”
看到“我今天”，预测“想”
看到“我今天想”，预测“吃”
看到“我今天想吃”，预测“火锅”
```

这个目标看起来很简单，但当训练数据足够大、模型参数足够多、网络结构足够强时，模型会逐渐学到语言、知识、推理模式、代码模式和对话模式。

所以可以先记住：

```text
LLM 的很多复杂能力，都是从“预测下一个 token”这个简单目标里长出来的。
```

### 6. 第四阶段：理解 Transformer 为什么重要

LLM 不是普通神经网络直接堆出来的，它通常建立在 Transformer 架构之上。

学习 Transformer 时，不要先陷入所有细节，可以先抓住几个问题：

- 模型如何知道一句话里不同词之间的关系？
- 模型如何判断当前 token 应该重点关注上下文中的哪些 token？
- 为什么长文本计算成本会变高？
- 为什么 GPT 类模型常用纯解码器架构？

Transformer 里最关键的思想是注意力机制。

可以先这样理解：

```text
注意力机制让模型在处理一个 token 时，动态判断上下文中哪些 token 更重要。
```

例如：

```text
小明把书放进书包，因为它太重了。
```

这里“它”更可能指“书”，而不是“书包”。注意力机制就是帮助模型在上下文里建立这种关联。

### 7. 第五阶段：理解训练和推理的区别

很多初学者会混淆训练阶段和回答阶段。

可以这样区分：

| 阶段 | 做什么 | 参数是否更新 |
| --- | --- | --- |
| 训练 | 用大量数据调整模型参数 | 会更新 |
| 推理 | 根据已有参数生成回答 | 不更新 |

用户平时调用 ChatGPT、Claude、Gemini 或本地模型，通常是在做推理。

也就是说：

```text
你问模型问题时，模型一般不会因为这次对话立刻修改自己的底层参数。
```

它只是利用已经训练好的参数，结合当前上下文，生成下一段输出。

### 8. 第六阶段：理解上下文窗口

LLM 不是无限记忆。

每次回答时，它能看到的内容受到上下文窗口限制。

上下文窗口可以理解为：

```text
模型当前这次推理最多能参考多少 token。
```

这会影响很多实际应用问题：

- 对话轮数太多，早期内容可能放不进去。
- 文档太长，需要切片、摘要或检索。
- Prompt 太长，会增加成本和延迟。
- 上下文里无关信息太多，模型可能抓不住重点。

所以在 AI 应用开发中，不能只想着“把所有内容都塞给模型”。

更合理的思路是：

```text
把真正相关的信息，以清晰结构放进上下文。
```

### 9. 第七阶段：把 RAG、微调和 Agent 放回原理主线

理解 LLM 原理后，再看 RAG、微调和 Agent，会更清楚它们分别在解决什么问题。

| 技术 | 解决的问题 |
| --- | --- |
| RAG | 模型不知道外部知识或私有数据时，先检索资料再回答 |
| 指令微调 | 让模型更会按照人的指令做事 |
| 奖励模型 / 强化学习 | 让模型输出更符合人类偏好 |
| Agent | 让模型不只生成文本，还能规划步骤、调用工具、执行任务 |
| 多模态 | 把文本、图像、音频、视频等信息统一进模型处理流程 |

RAG 不是强化学习，也不是直接训练模型参数。

它更像是：

```text
模型回答前，先从外部知识库查资料，再把资料放进上下文中。
```

所以 RAG 的关键不在于“模型变聪明了”，而在于“模型看到的信息更准了”。

### 10. 推荐学习顺序

这一章可以按下面的顺序来学：

```text
语义向量
  ↓
参数和权重
  ↓
反向传播与梯度下降
  ↓
训练流程
  ↓
参数规模和算力
  ↓
Transformer
  ↓
自回归与 NTP
  ↓
指令微调与奖励机制
  ↓
上下文窗口
  ↓
RAG 与应用结合
  ↓
多模态扩展
```

对应到本章课程，可以分成几个学习阶段：

| 阶段 | 对应内容 | 学习重点 |
| --- | --- | --- |
| 基础直觉 | 02-08 | 向量、参数、反向传播、梯度下降 |
| 训练机制 | 09-15 | 训练流程、参数规模、算力需求 |

### 11. 学完后要能回答的问题

学完这一章，不要求你能手写一个完整大模型，但应该能清楚回答这些问题：

- LLM 为什么要把文字变成向量？
- 参数和权重到底在模型里起什么作用？
- 训练阶段和推理阶段有什么区别？
- 反向传播和梯度下降分别解决什么问题？
- 为什么 GPT 类模型可以通过预测下一个 token 来生成回答？
- Transformer 的注意力机制大概解决了什么问题？
- 为什么参数越多通常能力越强，但也会带来成本问题？
- 为什么上下文窗口会影响长对话效果？
- 为什么 RAG 能减少幻觉，但不能彻底消灭幻觉？
- 多模态模型是如何把图片、音频、视频也变成可计算信息的？

如果这些问题能讲清楚，就说明已经从“会用大模型”进入到“理解大模型工作方式”的阶段了。

### 12. 一句话总结

LLM 原理学习最重要的是抓住主线：

```text
文字变成 token，token 变成向量，向量进入神经网络，模型通过训练调整参数，推理时根据上下文不断预测下一个 token，最终生成回答。
```

先把这条链路想明白，再去看 Transformer、上下文、RAG、微调和多模态，学习压力会小很多。

## 02 神经网络如何用向量解决语义问题？

神经网络不能直接理解文字。

对计算机来说，文字本身只是符号。比如：

```text
猫
狗
苹果
银行
```

这些字看起来有含义，但计算机最开始看到的只是编码后的字符。它不知道“猫”和“狗”都是动物，也不知道“苹果”既可能是水果，也可能是公司。

所以 LLM 要解决语义问题，第一步不是直接“理解文字”，而是：

```text
把文字变成可以计算的数字。
```

这组数字通常就是向量。

### 1. 什么是向量？

向量可以先理解成：

```text
一组有顺序的数字。
```

例如：

```text
猫 = [0.21, -0.35, 0.88, 0.14, ...]
狗 = [0.19, -0.31, 0.81, 0.20, ...]
汽车 = [-0.72, 0.44, 0.05, -0.63, ...]
```

真实模型里的向量维度会更多，可能是几百维、几千维甚至更高。

但先不用纠结每个数字具体代表什么。学习阶段可以先抓住一个核心直觉：

```text
语义相近的词，向量距离通常更近。
语义差异大的词，向量距离通常更远。
```

也就是说，模型不是靠“文字长得像不像”理解语义，而是靠“向量在数学空间里的关系”理解语义。

### 2. 为什么不能只用编号表示文字？

一种很直观的做法是给每个词一个编号：

```text
猫 = 1
狗 = 2
汽车 = 3
苹果 = 4
```

但这种方式有个问题：编号本身没有语义。

如果只看编号，模型可能会误以为：

```text
狗 比 猫 大 1
汽车 比 猫 大 2
苹果 比 猫 大 3
```

这显然没有意义。

编号只能表示“它是谁”，不能表示“它和谁更像、和谁有关、适合出现在什么上下文里”。

所以神经网络需要的不是简单编号，而是更丰富的向量表示。

### 3. 向量为什么能表达语义？

向量能表达语义，是因为它可以把一个词放到很多维度上同时描述。

为了方便理解，可以假设有几个非常简化的维度：

| 词语 | 动物特征 | 食物特征 | 科技公司特征 | 交通工具特征 |
| --- | --- | --- | --- | --- |
| 猫 | 0.95 | 0.02 | 0.00 | 0.00 |
| 狗 | 0.93 | 0.01 | 0.00 | 0.00 |
| 苹果 | 0.00 | 0.82 | 0.70 | 0.00 |
| 汽车 | 0.00 | 0.00 | 0.05 | 0.94 |

真实模型不会用这么直白的字段，也不会有人手动写“动物特征”“食物特征”。这些特征是在训练中自动学出来的。

但这个例子能帮助理解：

```text
一个词不是只有一个标签，而是可以同时拥有很多语义特征。
```

比如“苹果”这个词，在不同上下文里可能偏向不同语义：

```text
我吃了一个苹果。        -> 更像水果
苹果发布了新手机。      -> 更像科技公司
```

模型通过上下文中的其他词，调整当前 token 的语义理解。

### 4. 文本进入模型的大致流程

一句话进入 LLM，通常不会直接作为完整句子被模型理解，而是会经过类似下面的过程：

```text
原始文本
  ↓
分词，切成 token
  ↓
每个 token 查到对应的向量
  ↓
向量进入神经网络
  ↓
神经网络根据上下文不断更新这些向量的含义
  ↓
输出对下一个 token 的预测
```

例如：

```text
我喜欢吃苹果
```

可能被拆成：

```text
我 / 喜欢 / 吃 / 苹果
```

每个 token 都会变成一个向量：

```text
我     -> 向量 A
喜欢   -> 向量 B
吃     -> 向量 C
苹果   -> 向量 D
```

这些向量不是孤立使用的。模型会把它们放在一起计算，判断它们之间的关系。

因为上下文里出现了“喜欢”“吃”，所以这里的“苹果”更可能指水果。

如果句子是：

```text
苹果发布了新款芯片
```

上下文里出现“发布”“新款”“芯片”，模型就会更倾向于把“苹果”理解成公司。

### 5. 神经网络如何利用向量做判断？

神经网络的工作，可以粗略理解为：

```text
输入向量
  ↓
经过很多层计算
  ↓
不断提取和组合特征
  ↓
得到更适合当前任务的表示
```

例如一开始，“苹果”的基础向量可能同时包含水果和公司的语义。

但进入具体上下文后，神经网络会结合周围 token 的向量进行计算：

```text
吃 + 苹果 -> 水果语义更强
发布 + 苹果 -> 公司语义更强
```

这说明 LLM 里的向量不是静态地放在那里不变。

更准确地说：

```text
token 初始向量提供基础语义，神经网络根据上下文不断更新它的语义表示。
```

这也是为什么同一个词在不同句子里，模型可以理解成不同意思。

### 6. 向量之间可以计算关系

向量的好处是可以做数学计算。

模型可以计算两个向量之间的相似度，例如：

```text
猫 和 狗：相似度高
猫 和 汽车：相似度低
医生 和 医院：关系强
程序员 和 代码：关系强
```

这种相似度不是靠人工规则写出来的，而是从大量文本中学出来的。

比如模型在训练时经常看到：

```text
猫会叫
狗会叫
猫是宠物
狗是宠物
猫和狗都是动物
```

久而久之，模型就会把“猫”和“狗”放到比较接近的位置。

它不是像字典一样记住一句定义，而是在大量上下文中学到：

```text
这些词经常出现在相似语境里，所以它们的语义可能接近。
```

### 7. 向量解决的是“可计算的语义”

人类理解语义，靠经验、感官、常识和语言环境。

神经网络理解语义，靠的是：

```text
把词、句子、图片、声音等信息变成向量，
再通过向量之间的关系进行计算。
```

所以向量解决的不是“让机器像人一样真正体验世界”，而是解决一个工程问题：

```text
怎样把语义变成模型可以计算、比较、组合和预测的东西。
```

这就是向量在 LLM 中的重要意义。

### 8. 句子也可以表示成向量

不仅词可以变成向量，一句话、一段文本、一篇文章也可以变成向量。

例如：

```text
我想买一台适合打游戏的电脑
```

可以变成一个整体语义向量。

这个向量大概会包含这些含义：

```text
购买意图
电脑产品
游戏场景
性能需求
```

所以在 RAG 或语义搜索里，我们经常会把用户问题和文档片段都变成向量，然后比较它们的相似度：

```text
用户问题向量
  ↓
和知识库里的文档向量做相似度匹配
  ↓
找到语义最相关的资料
```

这就是为什么向量数据库在 AI 应用里很常见。

它解决的不是关键词是否完全一致，而是语义是否接近。

比如用户问：

```text
怎么退钱？
```

知识库里可能写的是：

```text
退款流程说明
```

这两个文本没有完全使用同一个词，但语义接近。向量检索就有机会把它们匹配起来。

### 9. 向量不是万能的

向量很强，但它不是万能答案。

它也会遇到一些问题：

- 语义很接近但实际意图不同，模型可能混淆。
- 上下文信息不足时，模型可能误判词义。
- 训练数据里偏见很强时，向量空间也可能学到偏见。
- 专业领域知识不足时，向量相似不代表答案正确。
- 相似度高只代表“可能相关”，不代表“一定可信”。

所以在实际 AI 应用中，向量通常要和其他机制配合使用：

```text
向量检索
关键词检索
规则过滤
重排序
上下文组织
模型生成
人工确认
```

不能简单认为“用了向量就一定理解准确”。

### 10. 一句话总结

神经网络用向量解决语义问题，本质上是：

```text
把文字从人类可读的符号，转换成机器可计算的数字空间；
再通过向量距离、上下文关系和神经网络层层计算，得到对语义的理解和预测。
```

简单说：

```text
文字本身不能计算，向量让语义可以被计算。
```

## 03 神经网络的参数与权重设计

上一节讲到，文字会先变成向量，再进入神经网络计算。

接下来要理解一个关键问题：

```text
神经网络到底靠什么做判断？
```

答案就是：

```text
靠大量可学习的参数。
```

在神经网络里，参数不是普通配置项，而是模型内部用来处理输入、提取特征、建立关系、生成输出的一堆数字。

这些数字里最常见、最重要的一类就是权重。

### 1. 参数是什么？

参数可以先理解成：

```text
模型内部可以被训练改变的数字。
```

比如一个非常简化的神经网络可能有这样的计算：

```text
输出 = 输入 × 权重 + 偏置
```

这里的：

```text
权重
偏置
```

就是模型参数。

真实 LLM 里的参数会复杂很多，但本质仍然是大量数字矩阵：

```text
embedding 矩阵
注意力层里的 Q/K/V 权重矩阵
前馈神经网络里的权重矩阵
归一化层里的可学习参数
输出层里的权重矩阵
```

所以当我们说一个模型有“70 亿参数”“700 亿参数”，意思不是它存了 70 亿条知识，而是：

```text
这个模型内部有 70 亿或 700 亿个可学习的数字。
```

### 2. 权重是什么？

权重可以理解成：

```text
某个输入特征对输出结果的影响程度。
```

如果一个特征很重要，它对应的权重可能更大。

如果一个特征不重要，权重可能更小。

如果一个特征会反向影响判断，权重甚至可能是负数。

举一个简化例子，假设模型要判断一句话是不是在表达“想买电脑”。

它可能会关注这些特征：

| 特征 | 权重直觉 |
| --- | --- |
| 出现“买” | 增强购买意图 |
| 出现“电脑” | 增强商品类别判断 |
| 出现“多少钱” | 增强购买或询价意图 |
| 出现“修电脑” | 可能不是购买，而是维修需求 |
| 出现“电影里的电脑特效” | 可能只是讨论内容，不是购买 |

这不是说真实模型里有人工写好的这些规则。

真实情况是：

```text
模型通过大量训练样本，自己学出哪些向量特征应该被放大，哪些应该被削弱。
```

这就是权重的作用。

### 3. 参数和权重是什么关系？

可以这样区分：

| 概念 | 含义 |
| --- | --- |
| 参数 | 模型中所有可以训练更新的数字 |
| 权重 | 参数中最核心的一类，主要负责特征变换和影响程度 |
| 偏置 | 另一类参数，可以理解成对输出结果做整体偏移 |

所以：

```text
权重属于参数，但参数不只包含权重。
```

在日常讨论 LLM 时，很多人会把“参数”和“权重”混着说，大多数场景下问题不大。但如果严格一点：

```text
参数 = 权重 + 偏置 + 其他可学习参数。
```

### 4. 参数不是人工写出来的知识

初学者很容易误解：

```text
模型知道很多知识，是不是因为工程师把知识写进了参数？
```

不是。

工程师不会手动写：

```text
参数 1 = 李白是唐代诗人
参数 2 = 猫是动物
参数 3 = Python 可以写后端
```

真实过程更像是：

```text
给模型大量文本
  ↓
模型尝试预测
  ↓
预测错了就计算误差
  ↓
通过反向传播和梯度下降调整参数
  ↓
重复很多很多次
```

训练完成后，知识不是以一条条文本形式存进去，而是分布在大量参数之间。

这叫分布式表示。

可以粗略理解为：

```text
一个知识点不是存放在某一个参数里，而是由很多参数共同影响出来。
```

这也是为什么我们很难直接打开一个模型参数文件，然后问：

```text
哪一个参数代表“北京是中国首都”？
```

通常找不到这样一个单独参数。

### 5. 神经网络设计的是结构，不是每个权重的具体值

“参数与权重设计”容易让人误以为工程师会逐个设计权重数值。

实际不是这样。

人类主要设计的是：

```text
网络有多少层
每层有多宽
向量维度是多少
注意力头有多少个
前馈网络放大多少倍
哪些层之间如何连接
使用什么激活函数
使用什么训练目标
```

这些通常叫模型结构或超参数。

而具体权重数值主要靠训练学出来。

可以这样类比：

```text
工程师设计学校、课程和考试方式；
学生具体学到什么水平，要靠大量训练。
```

对应到模型里：

```text
结构设计决定模型有多大的学习空间。
训练过程决定参数最终变成什么样。
```

### 6. 一个神经元如何使用权重？

为了理解权重，可以看一个极简神经元。

假设输入有三个特征：

```text
x1 = 是否出现“买”
x2 = 是否出现“电脑”
x3 = 是否出现“维修”
```

对应三个权重：

```text
w1 = 0.8
w2 = 0.7
w3 = -0.5
```

模型可能做这样的计算：

```text
得分 = x1 × w1 + x2 × w2 + x3 × w3 + 偏置
```

如果一句话是：

```text
我想买一台电脑
```

那么“买”和“电脑”相关特征会增强结果。

如果一句话是：

```text
我想维修电脑
```

“维修”这个特征可能会把“购买意图”的得分拉低。

真实 LLM 不会用这么简单的三个特征，但原理上仍然是：

```text
输入向量经过权重矩阵变换，不断形成新的特征表示。
```

### 7. 权重矩阵为什么重要？

LLM 处理的不是单个数字，而是高维向量。

所以真实计算通常不是：

```text
一个数字 × 一个权重
```

而是：

```text
一个向量 × 一个权重矩阵
```

可以简单理解为：

```text
权重矩阵负责把输入向量变换成新的向量。
```

例如：

```text
原始 token 向量
  ↓
经过注意力层权重矩阵
  ↓
得到带有上下文关系的新向量
  ↓
经过前馈网络权重矩阵
  ↓
得到更复杂的语义特征
```

每一层都在做类似的事情：

```text
把上一层的表示，变成更适合下一步判断的新表示。
```

这就是为什么深度神经网络要有很多层。

### 8. 参数规模为什么会影响模型能力？

参数越多，模型通常有更强的表达能力。

可以粗略理解为：

```text
参数越多，模型可以表示的模式越复杂。
```

小模型可能只能学到比较简单的语言规律：

```text
常见词语搭配
简单问答
基础分类
```

大模型有机会学到更复杂的模式：

```text
长距离上下文关系
多步骤推理模式
代码结构
专业知识关联
跨语言表达
复杂指令遵循
```

但参数越多不代表一定越聪明。

模型能力还取决于：

- 数据质量
- 数据规模
- 模型结构
- 训练目标
- 训练策略
- 对齐方法
- 推理时的上下文组织

所以更准确的说法是：

```text
参数规模提供能力上限，训练质量决定能不能把这个上限发挥出来。
```

### 9. 参数太少和参数太多分别有什么问题？

参数太少，模型表达能力不够，容易欠拟合。

也就是：

```text
模型太简单，学不会复杂规律。
```

例如让一个很小的模型去完成复杂法律分析、长代码生成、多轮推理，它可能能力不够。

参数很多，模型表达能力更强，但也会带来新的问题：

- 训练成本更高
- 推理成本更高
- 显存占用更大
- 延迟更高
- 更依赖高质量数据
- 更容易记住训练数据中的噪声
- 部署门槛更高

所以实际工程里，不是永远选择最大模型，而是要在效果、成本、速度、部署条件之间权衡。

例如：

```text
简单分类任务：小模型可能足够
复杂 Agent 规划：需要更强模型
本地离线应用：要考虑显存和推理速度
企业知识库问答：可能用中等模型 + RAG 更划算
```

### 10. 训练阶段参数会更新，推理阶段参数通常固定

这是理解 LLM 很重要的一点。

训练阶段：

```text
模型预测
  ↓
计算误差
  ↓
反向传播
  ↓
更新参数
```

推理阶段：

```text
输入 prompt
  ↓
使用已经训练好的参数计算
  ↓
生成回答
```

推理时，模型一般不会修改自己的底层参数。

比如你问模型：

```text
请记住我的名字叫小明。
```

在当前对话里，模型可以通过上下文“记住”这件事。

但这通常不是把“小明”写进模型参数，而是：

```text
这条信息还在当前上下文窗口里，所以模型回答时能看到。
```

如果要长期记忆，一般需要外部存储、数据库、用户画像系统或重新训练/微调，而不是普通聊天时自动改参数。

### 11. 参数、超参数和配置不要混淆

实际学习中还会看到“超参数”这个词。

它和参数不是一回事。

| 类型 | 谁决定 | 举例 |
| --- | --- | --- |
| 参数 | 训练学出来 | 权重、偏置、归一化参数 |
| 超参数 | 人或训练框架预先设定 | 层数、隐藏维度、学习率、batch size |
| 普通配置 | 应用运行时设置 | temperature、max tokens、top_p |

例如：

```text
模型某一层的权重 = 参数
模型有多少层 = 超参数
调用 API 时设置 temperature = 普通推理配置
```

这些概念如果混在一起，后面学训练、微调和推理优化时会很乱。

### 12. LLM 里的参数大概分布在哪里？

以 Transformer 类模型为例，参数主要分布在几个地方：

| 位置 | 作用 |
| --- | --- |
| token embedding | 把 token ID 转成向量 |
| position embedding / 位置相关参数 | 帮模型理解顺序信息 |
| attention 权重矩阵 | 计算 token 之间应该关注谁 |
| feed-forward 权重矩阵 | 对每个位置的语义表示做更复杂变换 |
| layer norm 参数 | 稳定每层的数值分布 |
| 输出层权重 | 把隐藏状态转换成下一个 token 的概率分布 |

不用现在就记住每个矩阵的细节。

先建立一个感觉就够了：

```text
LLM 的每一层都包含大量矩阵参数，这些矩阵共同决定输入如何一步步变成输出。
```

### 13. 为什么参数可以承载知识和能力？

模型训练时会看到大量文本模式。

例如：

```text
巴黎是法国的首都。
法国首都是巴黎。
Paris is the capital of France.
去法国旅游常会到巴黎。
```

这些句子反复出现后，模型会通过参数调整，逐渐形成某种稳定的关联。

以后看到：

```text
法国的首都是哪里？
```

模型就更容易预测出：

```text
巴黎
```

但这里要注意：

```text
模型不是像数据库一样精确存储事实，而是在参数中形成统计关联和语义模式。
```

这也解释了为什么模型会幻觉。

当模型没有足够可靠的知识，或者上下文暗示错误时，它仍然会根据参数里的模式生成“看起来合理”的内容。

### 14. 一句话总结

神经网络的参数与权重，本质上是：

```text
模型内部用于处理向量、提取特征、建立关系、生成预测的大量可学习数字。
```

人类设计的是网络结构和训练方式，模型通过训练自己学出具体权重。

简单说：

```text
结构决定模型怎么学，参数记录模型学到了什么，权重决定输入如何影响输出。
```

## 04 什么是反向传播？

反向传播是神经网络训练中最核心的机制之一。

它解决的问题可以先用一句话理解：

```text
模型预测错了以后，怎么知道每个参数应该往哪个方向调整。
```

神经网络里有大量参数。一次预测出错时，不能只说“模型错了”，还要进一步判断：

```text
哪些参数对这个错误影响更大？
这些参数应该调大还是调小？
每个参数应该调整多少？
```

反向传播就是用来回答这些问题的。

### 1. 先理解完整训练流程

神经网络训练通常不是一步完成的，而是不断重复下面这个流程：

```text
输入训练数据
  ↓
前向传播，模型给出预测
  ↓
计算预测结果和标准答案之间的误差
  ↓
反向传播，计算每个参数对误差的影响
  ↓
梯度下降，根据影响程度更新参数
  ↓
继续下一轮训练
```

这里面有三个概念容易混：

| 概念 | 作用 |
| --- | --- |
| 前向传播 | 从输入一路算到输出，得到模型预测 |
| 损失函数 | 衡量模型预测和正确答案差多少 |
| 反向传播 | 从误差反推每个参数对错误的贡献 |

所以反向传播不是完整训练过程的全部，它只是训练过程中的一个关键环节。

### 2. 什么是前向传播？

前向传播就是模型正常做一次预测。

例如我们给模型一句话：

```text
床前明月光，疑是地上
```

训练目标可能是让模型预测下一个 token：

```text
霜
```

模型根据当前参数进行计算，可能输出一个概率分布：

```text
霜：0.40
雪：0.20
光：0.10
其他：0.30
```

如果正确答案是“霜”，但模型只给了 0.40 的概率，说明它还不够确定。

这一步从输入到输出的计算过程，就叫前向传播。

```text
输入 -> 一层层神经网络计算 -> 输出预测结果
```

### 3. 什么是损失？

损失可以理解成：

```text
模型这次预测错得有多离谱。
```

如果正确答案是“霜”，模型给“霜”的概率越高，损失越小。

如果模型给“霜”的概率很低，而把概率给了别的 token，损失就更大。

例如：

```text
正确答案：霜

模型 A：
霜：0.90
雪：0.05
其他：0.05
损失较小

模型 B：
霜：0.10
雪：0.70
其他：0.20
损失较大
```

损失函数就是把“错得有多离谱”变成一个数字。

反向传播需要依赖这个数字，因为它要从这个错误出发，反推参数应该怎么调整。

### 4. 反向传播到底“反向”在哪里？

前向传播是：

```text
输入层
  ↓
中间层
  ↓
输出层
  ↓
得到预测
```

反向传播是：

```text
损失
  ↓
输出层
  ↓
中间层
  ↓
更前面的层
  ↓
计算各层参数的梯度
```

它的“反向”，不是说模型把句子倒着读，而是说：

```text
误差信号从输出端往输入端方向传回去。
```

模型先知道最终结果错了，然后一层层往回分析：

```text
最后一层哪些权重导致了这个错误？
再前一层哪些权重影响了最后一层？
更前面的层又承担了多少责任？
```

这个过程就像从结果倒查原因。

### 5. 梯度是什么？

反向传播最后会算出梯度。

梯度可以先理解成：

```text
某个参数变化一点点，会让损失变大还是变小，以及影响有多大。
```

如果一个参数变大，会让损失变小，说明它可能应该调大。

如果一个参数变大，会让损失变大，说明它可能应该调小。

如果一个参数变化对损失几乎没影响，说明这次训练样本里它不太关键。

所以梯度回答的是：

```text
参数应该往哪个方向动。
```

而梯度下降回答的是：

```text
按照梯度提示，实际把参数更新一下。
```

这就是反向传播和梯度下降的区别。

### 6. 用一个生活例子理解反向传播

假设一个团队做项目，最后项目失败了。

如果只说：

```text
项目失败了，大家都改进一下。
```

这没有用，因为不知道谁的问题是什么。

更有效的复盘是：

```text
需求理解偏了多少？
设计方案影响了多少？
开发实现出错多少？
测试遗漏了多少？
沟通延迟影响了多少？
```

这就像反向传播。

它不是只看到最终错误，而是把错误往回拆，分配到每个环节。

对应到神经网络里：

```text
最终预测错了
  ↓
分析输出层参数的责任
  ↓
分析中间层参数的责任
  ↓
分析更前面层参数的责任
  ↓
得到每个参数的调整方向
```

### 7. 用一个极简模型理解反向传播

假设模型只有一个参数：

```text
预测结果 = 输入 × 权重
```

输入是：

```text
2
```

正确答案是：

```text
10
```

如果当前权重是：

```text
3
```

那么模型预测：

```text
2 × 3 = 6
```

预测结果 6 比正确答案 10 小。

这说明当前权重可能太小了，需要调大。

反向传播要做的事情就是：

```text
根据“预测偏小”这个错误，计算权重应该增加多少。
```

真实神经网络有很多层、很多矩阵、很多非线性计算，反向传播就是用系统化的方法，把这种“应该怎么调”的信息传回到所有相关参数上。

### 8. 为什么深层网络必须要反向传播？

如果模型只有一个参数，人工判断怎么调还比较容易。

但真实神经网络里可能有：

```text
几百万参数
几十亿参数
几百亿参数
```

而且这些参数不是单独起作用的，而是层层组合：

```text
第一层的输出影响第二层
第二层的输出影响第三层
第三层的输出影响最终预测
```

最终预测错了，不能简单地知道某一个参数该怎么改。

反向传播的价值就在这里：

```text
它可以高效计算大量参数对最终损失的影响。
```

没有反向传播，深度神经网络几乎无法进行大规模训练。

### 9. 反向传播不是“人教模型知识”

反向传播经常被误解成：

```text
人告诉模型哪里错了，然后把正确知识写进去。
```

不是。

更准确地说：

```text
人或数据提供训练样本和目标答案；
损失函数计算模型错多少；
反向传播计算参数责任；
优化器更新参数。
```

也就是说，反向传播并不直接理解“知识是什么”。

它只是在数学层面计算：

```text
为了让下次损失更小，参数应该如何调整。
```

模型的知识和能力，是在大量这样的更新中逐渐形成的。

### 10. 反向传播和数据标注的关系

数据标注不是反向传播。

标注数据的作用是提供参考答案。

例如：

```text
输入：这家餐厅太难吃了
标签：负面评价
```

模型先预测，假设它预测成：

```text
正面评价
```

然后损失函数发现预测和标签不一致。

接着反向传播根据这个错误计算参数梯度。

所以这几个步骤的关系是：

```text
标注数据提供正确答案
损失函数衡量预测差距
反向传播计算参数责任
梯度下降更新参数
```

标注数据是训练材料，反向传播是训练算法的一部分。

### 11. 反向传播在 LLM 训练中怎么体现？

LLM 的预训练通常会做下一个 token 预测。

例如训练文本是：

```text
我喜欢学习人工智能
```

模型会学习：

```text
看到“我”，预测“喜欢”
看到“我喜欢”，预测“学习”
看到“我喜欢学习”，预测“人工”
看到“我喜欢学习人工”，预测“智能”
```

如果模型预测错了，就会产生损失。

反向传播会把这个损失传回模型内部：

```text
输出层
  ↓
Transformer 后面的层
  ↓
Transformer 前面的层
  ↓
embedding 相关参数
```

然后优化器更新相关参数，让模型下次更有可能预测正确。

这件事会在海量文本上重复很多次。

最终模型就逐渐学到了语言规律、知识关联、代码模式和指令模式。

### 12. 反向传播会带来哪些问题？

反向传播很强，但训练深层神经网络时也会遇到一些问题：

- 梯度太小，前面层学不动，叫梯度消失。
- 梯度太大，参数更新过猛，叫梯度爆炸。
- 学习率太大，模型可能来回震荡。
- 学习率太小，训练会非常慢。
- 数据质量差，模型会把错误模式也学进去。
- 模型太大，反向传播需要大量显存和算力。

这也是为什么现代 LLM 训练需要很多工程技巧，例如归一化、残差连接、学习率调度、梯度裁剪、混合精度训练等。

这些细节可以后面再深入，先记住：

```text
反向传播负责算方向，但训练稳定性还需要很多配套设计。
```

### 13. 一句话总结

反向传播的本质是：

```text
从模型输出的错误出发，沿着网络结构一层层往回计算每个参数对错误的影响。
```

它告诉模型：

```text
哪些参数该调大，哪些参数该调小，以及大概要调多少。
```

简单说：

```text
前向传播负责做题，损失函数负责判卷，反向传播负责追责，梯度下降负责改错。
```

## 05 如何解决过拟合的问题？

过拟合是机器学习里非常常见的问题。

它可以先用一句话理解：

```text
模型在训练数据上表现很好，但换到新数据上表现变差。
```

也就是说，模型不是没有学会，而是学得太死了。

它把训练数据里的细节、噪声、偶然规律都记住了，但没有真正学到能泛化到新场景的规律。

### 1. 什么是过拟合？

假设我们训练一个模型判断图片里是不是猫。

训练集中猫的图片大多是：

```text
白色背景
猫在画面中间
光线很亮
照片比较清晰
```

如果模型过拟合，它可能学到的不是“猫的本质特征”，而是：

```text
白色背景 + 中间有毛茸茸物体 + 光线很亮 = 猫
```

结果一旦换成：

```text
黑色背景里的猫
侧脸猫
模糊照片里的猫
猫只露出一部分
```

模型就容易判断错。

这说明它在训练集上记得很熟，但泛化能力不够。

### 2. 过拟合和欠拟合的区别

欠拟合和过拟合是两个相反的问题。

| 问题 | 表现 | 原因直觉 |
| --- | --- | --- |
| 欠拟合 | 训练集和测试集都表现差 | 模型太弱，连基本规律都没学会 |
| 过拟合 | 训练集表现好，测试集表现差 | 模型太死，记住了训练集细节 |

可以这样理解：

```text
欠拟合：学不会。
过拟合：背答案。
泛化好：理解规律。
```

训练模型真正追求的不是训练集分数最高，而是：

```text
在没见过的新数据上也表现稳定。
```

### 3. 为什么会过拟合？

过拟合通常不是单一原因造成的，而是多个因素叠加。

常见原因包括：

- 训练数据太少。
- 训练数据太单一。
- 数据里噪声太多。
- 标签质量差。
- 模型参数太多，表达能力太强。
- 训练时间太长。
- 训练集和真实使用场景差异太大。
- 评估方式不合理，只盯着训练集效果。

可以用一个简单关系理解：

```text
模型能力很强 + 数据不足或质量差 = 容易过拟合。
```

大模型参数很多，表达能力很强，所以更需要大量高质量、多样化的数据和严格评估。

### 4. 解决思路一：增加数据量

最直接的办法是增加训练数据。

如果模型只看过很少的样本，它很容易记住这些样本的细节。

如果样本足够多，模型更容易学到稳定规律。

例如猫图片分类：

```text
只看 100 张猫图：容易记住背景、姿势、光线
看 100 万张猫图：更容易学到猫本身的特征
```

对 LLM 来说也类似。

如果训练数据覆盖不够，模型可能只学到某些固定表达或局部模式。

更大、更广、更干净的数据集，可以让模型看到更多语言现象、知识场景和表达方式，从而减少只记住局部样本的风险。

### 5. 解决思路二：提高数据多样性

只增加数量还不够，数据还要多样。

如果数据很多但类型单一，模型仍然可能过拟合到某种固定模式。

例如训练客服模型时，如果训练数据全是：

```text
退款问题
```

那模型遇到物流、发票、账户、安全、售后等问题时，就可能表现不稳定。

更好的数据应该覆盖：

```text
不同问题类型
不同表达方式
不同用户语气
不同业务场景
不同边界情况
不同难度样本
```

对 LLM 来说，数据多样性尤其重要。

因为它需要处理开放语言输入，而不是只处理固定格式表单。

### 6. 解决思路三：清洗数据和降低噪声

过拟合不只是“记住正确样本”，也可能是“记住错误样本”。

如果训练数据里有大量噪声，例如：

- 错误答案
- 重复内容
- 低质量文本
- 互相矛盾的标注
- 格式混乱的数据
- 广告、垃圾内容、无意义文本

模型就可能把这些东西也学进去。

所以数据清洗很重要。

对 LLM 训练来说，数据质量经常比单纯数据规模更关键。

可以这样理解：

```text
高质量数据帮助模型学习规律。
低质量数据会让模型记住噪声。
```

### 7. 解决思路四：划分训练集、验证集和测试集

解决过拟合，不能只看训练集表现。

通常会把数据分成：

| 数据集 | 作用 |
| --- | --- |
| 训练集 | 用来训练模型、更新参数 |
| 验证集 | 用来观察模型是否开始过拟合，辅助调参 |
| 测试集 | 用来最终评估模型在未见数据上的效果 |

如果训练集效果越来越好，但验证集效果开始变差，就说明模型可能开始过拟合。

例如：

```text
训练集准确率：95% -> 98% -> 99%
验证集准确率：92% -> 91% -> 88%
```

这时继续训练虽然能让训练集更好，但真实泛化能力反而下降。

所以评估模型时一定要看未参与训练的数据。

### 8. 解决思路五：早停

早停就是：

```text
当验证集效果不再提升，甚至开始下降时，停止训练。
```

因为继续训练下去，模型可能不是在学习通用规律，而是在记训练集细节。

训练过程可能是这样：

```text
前期：训练集变好，验证集也变好，说明模型在学习规律
中期：训练集继续变好，验证集提升变慢
后期：训练集继续变好，验证集开始变差，说明可能过拟合
```

早停就是在合适的位置停下来。

它解决的是：

```text
不要让模型为了训练集分数继续死记硬背。
```

### 9. 解决思路六：正则化

正则化是一类约束模型的方法。

它的核心思想是：

```text
不要让模型为了拟合训练集，把参数调整得过于极端。
```

常见正则化方式包括：

```text
L1 正则化
L2 正则化
weight decay
dropout
```

学习 LLM 原理时，不需要一开始掌握所有数学细节。

先理解直觉就够了：

```text
正则化会给模型加一些限制，让它别太依赖训练集里的某些偶然特征。
```

比如 dropout 会在训练时随机“关掉”一部分神经元，让模型不能过度依赖某几个固定路径。

这有点像训练一个团队：

```text
不能每次都只靠某一个人解决问题，而是让整体能力更稳。
```

### 10. 解决思路七：控制模型复杂度

模型越复杂，表达能力越强，也越容易记住训练集细节。

如果任务很简单，数据又不多，用一个超大模型可能反而容易过拟合。

可以通过这些方式控制复杂度：

- 减少层数。
- 减少隐藏维度。
- 减少参数量。
- 使用更小的模型。
- 限制训练轮数。
- 只微调部分参数。

在实际工程里，模型不是越大越好，而是要看任务、数据和成本。

例如：

```text
小数据集 + 简单分类任务：小模型可能更稳
复杂推理任务 + 大量高质量数据：大模型更有优势
企业知识库问答：中等模型 + RAG 可能比盲目微调大模型更合适
```

### 11. 解决思路八：数据增强

数据增强就是在不改变核心语义的前提下，制造更多变化。

图像任务里常见做法包括：

```text
旋转
裁剪
缩放
改变亮度
加入噪声
```

文本任务里也可以做类似增强，例如：

```text
同义改写
改变句式
加入不同语气
构造边界样本
生成反例
扩展多轮对话场景
```

数据增强的目的不是简单堆数量，而是让模型看到同一个语义在不同表达下的样子。

这样模型就不容易只记住某个固定表达。

### 12. 过拟合在 LLM 微调中尤其要小心

在 LLM 应用开发中，很多人会做微调。

微调时过拟合很常见，因为企业或个人手里的数据集通常不大。

例如只用几百条客服对话微调一个模型，可能会出现：

- 模型只会某几种固定话术。
- 遇到新问题不会灵活处理。
- 输出风格变得僵硬。
- 把训练样本里的错误格式也学进去。
- 对少数样本里的特殊表达过度敏感。

所以微调时要特别注意：

```text
数据质量
数据多样性
验证集表现
训练轮数
学习率
是否真的需要微调
```

很多场景并不一定要微调。

如果问题主要是模型缺少企业知识，通常优先考虑：

```text
RAG
Prompt 设计
上下文工程
工具调用
```

因为这类问题往往不是“模型能力不够”，而是“模型没看到正确资料”。

### 13. 如何判断是否过拟合？

可以从几个信号判断：

- 训练集效果很好，验证集或测试集效果明显差。
- 模型对训练样本回答很准，对新问题不稳定。
- 模型输出大量训练集中常见的固定话术。
- 换一种问法后效果明显下降。
- 模型记住了训练样本里的错误或隐私信息。
- 继续训练时训练损失下降，但验证损失上升。

最典型的判断方式是看训练集和验证集的差距：

```text
训练集好，验证集也好：泛化较好
训练集差，验证集也差：欠拟合
训练集好，验证集差：过拟合
```

### 14. 解决过拟合的整体策略

可以把解决过拟合的方法分成四类：

| 方向 | 方法 |
| --- | --- |
| 数据 | 增加数据、提高多样性、清洗噪声、数据增强 |
| 模型 | 控制参数规模、选择合适结构、只微调必要部分 |
| 训练 | 早停、正则化、调整学习率、减少训练轮数 |
| 评估 | 划分验证集和测试集，持续观察泛化效果 |

所以解决过拟合，不是单靠一个技巧，而是一个完整训练策略。

### 15. 一句话总结

过拟合的本质是：

```text
模型太贴合训练数据，记住了细节和噪声，却没有学到能泛化的新规律。
```

解决过拟合的核心是：

```text
让模型少背训练集，多学通用规律。
```

简单说：

```text
训练集表现好不代表模型真的好，能在新数据上稳定表现才是真的好。
```

## 06 数据标注是反向传播吗？

数据标注不是反向传播。

这是两个完全不同层面的概念。

可以先用一句话区分：

```text
数据标注是准备训练材料，反向传播是模型内部更新参数的算法过程。
```

数据标注告诉模型“什么是参考答案”。

反向传播负责在模型预测错之后，计算参数应该怎么调整。

### 1. 什么是数据标注？

数据标注就是给原始数据加上有用的标签、答案或结构化信息。

例如情感分类任务：

```text
原始文本：这家餐厅太难吃了
标注结果：负面评价
```

图片分类任务：

```text
原始图片：一张猫的照片
标注结果：猫
```

问答任务：

```text
问题：法国的首都是哪里？
标注答案：巴黎
```

客服任务：

```text
用户问题：我想退货怎么办？
标注答案：引导用户进入退货流程，并说明退货条件
```

所以数据标注的本质是：

```text
给训练样本提供参考答案或监督信号。
```

它发生在训练之前，属于数据准备环节。

### 2. 什么是反向传播？

反向传播发生在模型训练过程中。

它的作用是：

```text
根据预测结果和参考答案之间的误差，计算每个参数该怎么调整。
```

训练时模型会先做一次预测。

例如：

```text
输入：这家餐厅太难吃了
正确标签：负面评价
模型预测：正面评价
```

模型预测错了。

接下来会发生几件事：

```text
损失函数计算错了多少
  ↓
反向传播计算哪些参数导致了错误
  ↓
梯度下降根据梯度更新参数
```

所以反向传播不是人工标注，也不是人工告诉模型知识。

它是模型训练时的一套数学计算过程。

### 3. 两者在训练流程中的位置

可以把完整流程拆开看：

```text
收集原始数据
  ↓
清洗数据
  ↓
数据标注，得到训练样本和参考答案
  ↓
模型读取训练样本
  ↓
前向传播，模型做预测
  ↓
损失函数，比较预测和参考答案
  ↓
反向传播，计算参数梯度
  ↓
优化器更新参数
  ↓
评估模型效果
```

这条链路里，数据标注在前面，反向传播在后面。

它们的关系是：

```text
没有标注数据，很多监督训练任务就没有明确参考答案。
没有反向传播，模型就很难根据错误自动更新参数。
```

一个负责提供答案，一个负责更新模型。

### 4. 用学生做题来类比

可以把训练模型类比成学生刷题。

数据标注像是：

```text
题目后面的标准答案。
```

例如：

```text
题目：1 + 1 = ?
标准答案：2
```

模型预测像是学生做题：

```text
学生答案：3
```

损失函数像是判卷：

```text
答案错了，差距是 1。
```

反向传播像是复盘责任：

```text
是加法规则没掌握？
是某一步计算方式错了？
哪些内部判断需要调整？
```

梯度下降像是改错：

```text
根据复盘结果，调整学习方式。
```

所以：

```text
标准答案不是复盘过程。
数据标注也不是反向传播。
```

### 5. 数据标注提供的是监督信号

监督学习里的“监督”，可以理解成训练时有参考答案。

例如：

| 任务 | 输入 | 标注 |
| --- | --- | --- |
| 情感分类 | 这家店太差了 | 负面 |
| 意图识别 | 我想查订单 | 查询订单 |
| 命名实体识别 | 马云创办了阿里巴巴 | 人名：马云，组织：阿里巴巴 |
| 图像分类 | 一张狗的图片 | 狗 |
| 问答训练 | 问题 | 标准回答 |

这些标注让模型知道：

```text
面对这个输入，期望输出应该是什么。
```

但标注本身不会自动改变模型参数。

只有把这些样本送进训练过程，经过前向传播、损失计算、反向传播和参数更新，模型才会真的学习。

### 6. 标注数据质量会影响反向传播方向

虽然数据标注不是反向传播，但它会影响反向传播。

因为反向传播是根据损失来计算梯度，而损失来自：

```text
模型预测结果 vs 标注答案
```

如果标注答案是错的，模型就会朝错误方向学习。

例如：

```text
文本：这家餐厅太难吃了
错误标注：正面评价
```

模型如果预测“负面评价”，本来是对的。

但因为标注错了，损失函数会认为模型错了。

然后反向传播会推动模型参数往“更容易预测正面评价”的方向更新。

这就会污染模型。

所以可以这样理解：

```text
标注质量决定训练信号是否可靠。
反向传播负责忠实地根据训练信号更新参数。
```

如果训练信号错了，反向传播也会认真地把错误学进去。

### 7. LLM 预训练一定需要人工标注吗？

不一定。

主流 LLM 的预训练通常使用自监督学习。

它的训练目标是：

```text
根据前面的 token，预测下一个 token。
```

这时不一定需要人工逐条标注答案，因为答案可以从原始文本中自动构造出来。

例如原始文本：

```text
我喜欢学习人工智能
```

可以自动变成训练样本：

```text
输入：我
答案：喜欢

输入：我喜欢
答案：学习

输入：我喜欢学习
答案：人工

输入：我喜欢学习人工
答案：智能
```

这里的“标注”不是人工写的，而是文本本身提供了下一个 token。

所以 LLM 预训练常说是自监督，因为：

```text
训练目标来自数据本身，而不是完全依赖人工标注。
```

但即使是自监督训练，仍然会使用反向传播更新参数。

### 8. 指令微调里的数据标注是什么？

LLM 预训练后，通常还需要指令微调。

指令微调会准备类似这样的数据：

```text
用户指令：帮我写一封请假邮件
理想回答：您好，我因身体不适需要请假一天...
```

或者：

```text
用户问题：解释一下什么是 RAG
理想回答：RAG 是检索增强生成...
```

这些“理想回答”就属于标注数据的一种。

它们告诉模型：

```text
面对人类指令时，应该用什么样的格式、语气和内容来回答。
```

训练时，模型会尝试生成回答。

如果生成内容和理想回答差距大，就会产生损失。

然后反向传播会根据这个损失更新参数，让模型以后更接近期望输出。

所以指令微调里的关系是：

```text
标注数据定义“好回答长什么样”
反向传播让模型逐步学会生成这种回答
```

### 9. 人类反馈和反向传播也不是一回事

有些训练流程会用到人类反馈，例如让人比较两个回答哪个更好。

例如：

```text
问题：如何学习 Python？

回答 A：先学语法，再做小项目，再读源码。
回答 B：Python 是一种编程语言。

人工偏好：A 更好
```

这个偏好数据也不是反向传播。

它只是提供一种训练信号：

```text
人类更喜欢什么样的回答。
```

后续可能会用这些偏好数据训练奖励模型，或者优化语言模型。

在这个训练过程中，仍然会用到损失函数、反向传播和参数更新。

所以：

```text
人工反馈是数据来源。
反向传播是参数更新算法。
```

### 10. 用户纠正模型，模型会立刻反向传播吗？

一般不会。

比如你在一次对话里告诉模型：

```text
你刚才说错了，我的项目用的是 FastAPI，不是 Flask。
```

模型后续在当前对话里可能会改口：

```text
好的，按 FastAPI 来说明。
```

但这通常不是模型底层参数被反向传播更新了。

更常见的情况是：

```text
这条纠正信息还在上下文窗口里，模型回答时能看到。
```

普通聊天中的纠正，通常只是改变当前上下文，不是现场训练模型。

如果要让模型长期学会某类新规则，通常需要：

```text
收集反馈数据
清洗和标注
加入训练集或评估集
重新训练、微调或做偏好优化
```

这时才可能真正更新模型参数。

### 11. 数据标注、Prompt 和 RAG 的区别

在 AI 应用里，这几个概念也容易混。

| 概念 | 作用 | 是否更新模型参数 |
| --- | --- | --- |
| 数据标注 | 准备训练样本和参考答案 | 本身不更新 |
| 反向传播 | 根据损失计算参数梯度 | 训练时参与更新 |
| Prompt | 在推理时给模型当前任务说明 | 不更新 |
| RAG | 检索资料放进上下文再回答 | 不更新 |
| 微调 | 用训练数据继续训练模型 | 会更新 |

例如：

```text
把客服问答整理成训练数据：数据标注
用这些数据训练模型：会用到反向传播
把客服知识库检索出来放进 prompt：RAG
临时告诉模型按某种格式回答：Prompt
```

这些方案解决的问题不同，不要混在一起。

### 12. 为什么很多人会把数据标注和反向传播混淆？

因为它们都出现在“模型学习”这个大流程里。

从外部看，好像是：

```text
给模型一些带答案的数据
  ↓
模型就学会了
```

但中间其实有很多步骤：

```text
带答案的数据
  ↓
模型预测
  ↓
计算损失
  ↓
反向传播
  ↓
优化器更新参数
  ↓
多轮训练后能力变化
```

数据标注只是提供“带答案的数据”。

真正让参数发生变化的是训练过程中的反向传播和优化器更新。

### 13. 一句话总结

数据标注不是反向传播。

二者的关系是：

```text
数据标注提供正确答案或偏好信号；
反向传播根据模型预测和正确答案之间的误差，计算参数该怎么调整。
```

简单说：

```text
标注负责给答案，损失函数负责判差距，反向传播负责算责任，梯度下降负责改参数。
```

## 07 什么是梯度下降？解决了什么问题

梯度下降是训练神经网络时最核心的参数更新方法之一。

它解决的问题可以先用一句话理解：

```text
模型已经知道自己错了，也知道参数该往哪个方向调，接下来怎么一步步把错误变小。
```

前面讲反向传播时说过：

```text
反向传播负责计算梯度。
梯度下降负责根据梯度更新参数。
```

所以梯度下降不是用来“判断模型哪里错”，而是用来：

```text
沿着让损失变小的方向，一点点调整模型参数。
```

### 1. 先回到训练流程

完整训练流程可以这样看：

```text
输入训练样本
  ↓
前向传播，得到预测
  ↓
损失函数，计算预测错多少
  ↓
反向传播，计算每个参数的梯度
  ↓
梯度下降，根据梯度更新参数
  ↓
进入下一轮训练
```

这里每一步分工不同：

| 步骤 | 作用 |
| --- | --- |
| 前向传播 | 做一次预测 |
| 损失函数 | 衡量预测和答案的差距 |
| 反向传播 | 计算每个参数对损失的影响 |
| 梯度下降 | 按照梯度提示更新参数 |

可以简单记成：

```text
先算错多少，再算谁负责，最后改参数。
```

### 2. 什么是梯度？

梯度可以先理解成：

```text
损失函数在当前参数位置上的变化方向。
```

更直观一点：

```text
如果某个参数往某个方向变一点，损失会变大还是变小。
```

例如某个参数当前是：

```text
w = 3.0
```

如果把它调大，损失会变小，说明它应该往大的方向走。

如果把它调大，损失会变大，说明它应该往小的方向走。

梯度就是告诉模型：

```text
往哪里走，损失会上升最快。
```

而梯度下降要做的是反过来：

```text
往梯度的反方向走，让损失下降。
```

这也是为什么叫“下降”。

### 3. 用下山理解梯度下降

可以把模型训练想象成下山。

当前位置代表当前参数。

高度代表损失大小。

目标是走到更低的位置，也就是让损失更小。

```text
山越高：损失越大
山越低：损失越小
下山过程：不断调整参数
```

问题是，模型一开始不知道最低点在哪里。

它只能在当前位置观察坡度：

```text
哪个方向更陡？
往哪个方向走可以更快下降？
每次应该走多远？
```

梯度告诉它坡度方向。

梯度下降让它沿着下降方向走一步。

走完一步后，再重新计算当前位置的梯度，再走下一步。

```text
计算梯度
  ↓
向下降方向走一步
  ↓
重新计算梯度
  ↓
再走一步
  ↓
不断逼近低损失区域
```

### 4. 梯度下降解决了什么问题？

神经网络有大量参数。

训练模型就是要找到一组比较好的参数，让模型在训练任务上的损失尽量小。

问题是：

```text
参数数量太多，不可能靠人工一个个试。
```

例如一个大模型可能有：

```text
70 亿参数
700 亿参数
甚至更多参数
```

每个参数都可能影响输出。

人工不可能手动判断：

```text
第 1 个参数调大一点
第 2 个参数调小一点
第 3 个参数保持不变
……
```

梯度下降解决的就是这个问题：

```text
用数学方法自动调整大量参数，让模型损失逐步降低。
```

它让神经网络可以从错误中自动改进。

### 5. 梯度下降和反向传播的区别

这两个概念很容易混。

可以这样区分：

| 概念 | 解决的问题 |
| --- | --- |
| 反向传播 | 怎么算出每个参数的梯度 |
| 梯度下降 | 拿到梯度后怎么更新参数 |

反向传播像是：

```text
复盘谁导致了错误，以及每个人影响多大。
```

梯度下降像是：

```text
根据复盘结果，真正调整每个人的做法。
```

所以训练里一般是：

```text
先反向传播算梯度，再用梯度下降更新参数。
```

没有反向传播，梯度下降不知道怎么走。

没有梯度下降，算出梯度也不会真正改变参数。

### 6. 参数是怎么更新的？

梯度下降的更新逻辑可以简化成：

```text
新参数 = 旧参数 - 学习率 × 梯度
```

这里有两个关键点：

```text
梯度：告诉参数应该往哪个方向动
学习率：控制每次动多大步
```

为什么是减号？

因为梯度指向的是损失上升最快的方向。

我们希望损失下降，所以要往相反方向走。

也就是：

```text
朝梯度反方向更新参数。
```

### 7. 学习率是什么？

学习率可以理解成：

```text
每次更新参数时走多大一步。
```

学习率太大，会走得太猛。

可能出现：

```text
一步跨过最低点
来回震荡
损失不稳定
甚至训练崩掉
```

学习率太小，会走得太慢。

可能出现：

```text
训练速度很慢
很久都看不到效果
卡在不理想的位置附近
训练成本变高
```

所以学习率是训练中非常重要的超参数。

可以简单理解：

```text
学习率太大容易乱跑，学习率太小容易磨蹭。
```

### 8. 用一个极简例子理解

假设模型只有一个参数：

```text
w = 3
```

当前损失比较大。

反向传播算出梯度：

```text
gradient = 2
```

学习率是：

```text
learning_rate = 0.1
```

那么梯度下降更新参数：

```text
新 w = 旧 w - 学习率 × 梯度
新 w = 3 - 0.1 × 2
新 w = 2.8
```

也就是说，参数从 3 调整到了 2.8。

下一轮训练时，模型会用新的参数再预测一次。

如果损失仍然大，就继续：

```text
预测
算损失
算梯度
更新参数
```

大量重复之后，模型参数就会逐渐变得更合适。

### 9. 为什么不是一步到位？

很多人会问：

```text
既然知道方向，为什么不一步走到最优参数？
```

原因是神经网络的损失地形非常复杂。

它不是一个简单的平滑碗形，而是可能有：

```text
很多高低起伏
很多局部低点
很多平坦区域
很多陡峭区域
大量参数之间的复杂影响
```

模型只能根据当前位置的梯度，决定下一步怎么走。

每走一步，当前位置变了，梯度也会变。

所以训练通常是一个迭代过程：

```text
小步试探
不断修正
逐渐逼近更好的参数区域
```

这也是为什么训练大模型需要大量计算。

### 10. 批量梯度下降、小批量梯度下降和随机梯度下降

训练时，梯度可以基于不同数量的数据来算。

常见有几种方式：

| 方法 | 做法 | 特点 |
| --- | --- | --- |
| 批量梯度下降 | 每次用全部训练数据算梯度 | 稳定但很慢 |
| 随机梯度下降 | 每次用一个样本算梯度 | 快但波动大 |
| 小批量梯度下降 | 每次用一小批样本算梯度 | 实际最常用 |

大模型训练通常不会每次都用全部数据更新一次，因为成本太高。

更常见的是用 batch：

```text
取一批样本
  ↓
计算这一批的平均损失
  ↓
反向传播得到梯度
  ↓
更新一次参数
```

这就是小批量梯度下降的思路。

它在效率和稳定性之间做了折中。

### 11. 梯度下降和优化器是什么关系？

严格来说，梯度下降是一种基础优化思想。

实际训练中经常会用更高级的优化器，比如：

```text
SGD
Momentum
Adam
AdamW
```

这些优化器并不是推翻梯度下降，而是在梯度下降的基础上做改进。

例如它们可能会考虑：

- 当前梯度。
- 历史梯度趋势。
- 每个参数是否需要不同步长。
- 是否加入权重衰减。
- 如何让训练更稳定。

可以这样理解：

```text
梯度下降是基本方向盘。
Adam、AdamW 这类优化器是更智能的驾驶系统。
```

在 LLM 训练和微调里，AdamW 很常见。

### 12. 梯度下降会遇到什么问题？

梯度下降虽然核心思想简单，但实际训练会遇到很多问题。

常见问题包括：

- 学习率太大，训练震荡或发散。
- 学习率太小，训练太慢。
- 梯度消失，前面层几乎学不动。
- 梯度爆炸，参数更新过猛。
- 卡在局部较差区域。
- 数据噪声导致更新方向不稳定。
- batch 太小，梯度波动大。
- batch 太大，计算成本高且泛化可能受影响。

所以训练神经网络不是只写一句“梯度下降”就结束了。

还要配合：

```text
学习率调度
梯度裁剪
归一化
残差连接
合适 batch size
高质量数据
合理初始化
优化器选择
```

这些设计共同决定训练是否稳定。

### 13. 梯度下降在 LLM 训练中的意义

LLM 的训练目标通常是预测下一个 token。

例如：

```text
输入：床前明月光，疑是地上
正确答案：霜
```

模型一开始可能预测不准。

训练时会发生：

```text
模型预测下一个 token
  ↓
损失函数计算预测和正确 token 的差距
  ↓
反向传播计算所有相关参数的梯度
  ↓
优化器根据梯度更新参数
```

这个过程在海量文本上重复很多次。

最终模型就会逐渐学到：

```text
词语搭配
语法结构
事实关联
代码模式
推理模式
对话模式
```

所以 LLM 的能力，不是一次性写进模型里的，而是通过大量梯度更新逐渐形成的。

### 14. 梯度下降解决的不是所有问题

梯度下降只是训练方法的一部分。

它解决的是：

```text
如何根据损失自动更新参数。
```

但它不能单独解决所有问题。

例如：

- 数据错了，梯度下降会认真学习错误数据。
- 目标函数设计不好，模型会朝错误目标优化。
- 模型结构不合适，训练也很难达到好效果。
- 数据覆盖不足，模型泛化仍然会差。
- 推理时上下文组织不好，模型回答也可能差。

所以不要把模型训练理解成：

```text
只要梯度下降就能得到好模型。
```

更准确地说：

```text
好模型 = 好数据 + 好结构 + 好目标 + 好训练策略 + 好评估。
```

梯度下降只是其中非常关键的一环。

### 15. 一句话总结

梯度下降的本质是：

```text
根据梯度提示，沿着让损失下降的方向，一步步更新模型参数。
```
